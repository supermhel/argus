# FENGARDE v0.4.0 — opt-in auth, incident-report hook, three parser packs

**Tag:** `v0.4.0` · **Commit:** `153da94` · **Date:** 2026-07-12
**Full detail:** [`CHANGELOG.md`'s `[0.4.0]` section](../../CHANGELOG.md)

Everyone running FENGARDE before this release had a completely open pipeline —
no authentication anywhere. v0.4 makes it deployable beyond a trusted
localhost, and adds the three parser packs that came out of the project's
market/positioning research (AI-agent telemetry, industrial OT, and
low-code-automation logs).

## Highlights

- **Opt-in auth, everywhere it matters.** `FENGARDE_API_KEY` on the triage
  and inventory APIs, opt-in dashboard basic-auth, opt-in Redis `AUTH` — all
  default **off**, so a `docker compose up` on this tag behaves exactly like
  before unless you deliberately turn auth on. OpenSearch/Redis/Dashboards
  ports now bind to `127.0.0.1` by default.
- **The incident-report hook** — the open half of FENGARDE's regulatory-
  reporting story. `POST/GET /alerts/{id}/report` renders a generic markdown
  draft from real alert facts, structurally forced to `status: "draft"` with
  a mandatory disclaimer. The paid, legally-validated content layer
  (`fengarde-sec`) plugs in later through one frozen, documented seam
  (`contracts/reporting.md`) — nothing in this repo depends on it.
- **Three new parser packs:**
  - **MCP / AI-agent** tool-call logs → OCSF, with credential-access,
    tool-call-burst, and prompt-injection-indicator rules (labeled honestly
    as string-match heuristics, not a classifier).
  - **OPC UA / industrial OT** audit events (IEC 62541 Part 5) — maintenance-
    window, new-engineering-connection, and config-change rules.
  - **n8n automation-platform** audit logs — new-webhook-exposed and
    after-hours-modification rules.
- **Impossible-travel rule** — the first rule to actually consume the geo
  enrichment substrate that had shipped with zero consumers since v0.3.
- **Two real correctness bugs fixed along the way:** a stateful-rule counting
  bug that pooled events with no `group_by` value under a literal `"None"`
  bucket (and disagreed between the memory and Redis backends about what
  counts as a distinct value), and a cross-source rule-scoping bug where
  three rules keyed only on `class_uid` could mis-fire once a third source
  started emitting the same OCSF class.

## At this point in history

10 parsers, 17 rules, no multi-tenancy, no RBAC. Auth is a shared secret, not
per-user identity. See `v0.5.0`'s notes for everything that changed after
this tag.

## Upgrading from v0.3.0

No breaking changes. Auth is opt-in — nothing to migrate unless you choose to
turn it on.
