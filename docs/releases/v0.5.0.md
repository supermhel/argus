# FENGARDE v0.5.0 — MSP-grade: multi-tenancy, RBAC, NIS2, chaos-tested delivery

**Tag:** `v0.5.0` · **Commit:** `c3ec328` · **Date:** 2026-07-23
**Full detail:** [`CHANGELOG.md`'s `[0.5.0]` section](../../CHANGELOG.md)

This is the largest release in the project's history — everything from a
full pipeline hardening pass through the four criteria that gate a real
public launch (chaos-tested delivery, real proof artifacts, MSP-grade
multi-tenancy/RBAC, and a regulatory report template), plus a run of
continuous-improvement work (MITRE firing proof, observability, a second OT
source) that landed on top once those gates were already green.

**All four launch-gate criteria are green as of this tag.** That is a
technical fact, not a launch decision — nothing has been publicly announced;
see `SSOT.md` §5 for the honest, separate assessment of that question.

## Highlights

### The reliability guarantee this project is proudest of

**`make chaos`**: SIGKILLs each of the 5 pipeline services mid-replay across
40 independent attack scenarios, asserts zero lost and zero duplicated
alerts. **Live-verified passing** on a fresh Docker stack — at-least-once
delivery + Redis-backed window counters + deterministic `alert_id` genuinely
survive a SIGKILL of every stage. Getting an honest "yes" here took 4 real
harness-bug fixes across 4 runs; the write-up doesn't hide any of them.

### MSP-grade (the actual hard launch gate)

- **Multi-tenancy**: tenant-scoped OpenSearch indices, per-tenant rule
  enablement, isolation proven by an automated two-tenant test.
- **RBAC**: opt-in per-user identity, roles (read-only/analyst/admin),
  sessions, CSRF protection, a real dashboard login screen — off by default,
  every pre-existing deployment unaffected.
- **Versioned REST API** with a published OpenAPI 3.1 spec.
- **Outbound HMAC-signed webhooks**, delivered on their own consumer group so
  a slow receiver never delays indexing.
- **Entry-points plugin interface** — an external pip package can ship a
  parser or rule pack with no fork.
- **Ops lifecycle**: schema migration, backup/restore, versioned index-
  template migration, disk-usage guardrails.

### NIS2 public template layer

A deterministic German/English NIS2 Art. 23 / §32 BSIG incident-report
generator, additive on the existing report hook. Every field the alert data
can't support renders as an explicit `[ANALYST MUST PROVIDE]` placeholder —
never fabricated. States its own NIS2-vs-DORA scope caveat inline. Zero LLM,
zero paid dependency; the legally-validated layer stays a separate,
optional add-on.

### Proof artifacts, not just claims

- Real, reproducible bench numbers (`tools/fengarde_bench.py`).
- Supply-chain hardening: CodeQL's first live scan found 5 HIGH alerts, all
  fixed at the design level — 0 open on `main`. Scorecard alerts 54→19 via
  SHA/digest pinning (the rest are accepted, documented trade-offs).
  `pip-audit`, `docker-build`, and `actionlint` gates added.
- **mypy flipped from informational to blocking** after a re-measure found
  47 real findings (not the stale "20" a comment claimed) — all fixed.
- **Mutation testing (mutmut)**, first run ever in this repo: 142 mutants on
  the session-store code, 72% kill rate, survivors disclosed not hidden.

### Continuous-track work that landed on top

- **MITRE empirical firing check**: every MITRE-tagged rule is now proven to
  actually fire on its own producer fixture through the live detection
  engine, not just declared via a `mitre:` tag. 26/26 fire.
- **Prometheus + Grafana observability**, opt-in compose profile,
  live-verified. OpenTelemetry tracing stays explicitly out of scope for
  now — an ADR aspiration, not silently rolled into "done."
- **Modbus/TCP protocol-anomaly detector** — a second OT source, deliberately
  scoped as a protocol-anomaly detector over the public function-code table
  rather than a vendor-log parser, since Modbus has no audit-log format to
  parse.

### A repo-wide adversarial bug hunt, not just a PR review

Six real bugs found and fixed, each independently verified to actually fail
without its fix (revert/run/restore cycle): a cross-tenant window-counter
pooling bug (and a follow-up review that caught the same bug still lurking
in `alert_id` generation after the first fix), an AD parser skipping the
type-guards every sibling parser already had, a tenant-gate bypass on aged-
out alerts, an unbounded/unlocked rate limiter, a tar-extraction
symlink-escape, and unvalidated `tenant_id` reaching an OpenSearch index name.

## At this point in history

16 parsers, 27 rules. Multi-tenancy and RBAC are real and tested. The chaos
gate passes live. Full detail — including every disclosed limitation this
release does *not* fix (OTel tracing, S7/PROFINET parser, live-stack load
testing, per-tenant retention override) — is in `SSOT.md`.

## Upgrading from v0.4.0

Multi-tenancy, RBAC, webhooks, and plugins are all opt-in — an existing
v0.4.0 deployment that sets none of the new env vars is byte-for-byte
unaffected. See `docs/ops-lifecycle.md` for the schema-migration and
index-template-migration tooling if you do opt in.
